11. Análisis mediante conjuntos PRIMERO y SIGUIENTE

Para realizar el análisis sintáctico LL(1), se construyen los conjuntos PRIMERO y SIGUIENTE de las principales producciones de la gramática. Estos conjuntos permiten decidir qué regla aplicar según el token de entrada.

Gramática utilizada


<programa> ::= <sentencia> <programa> | ε

<sentencia> ::= <declaracion>
              | <asignacion>
              | <impresion>
              | <condicional>
              | <bucle_for>
              | <bucle_while>

<declaracion> ::= <tipo_variable> IDENTIFICADOR "=" <valor> ";"

<tipo_variable> ::= "let" | "var" | "const"

<asignacion> ::= IDENTIFICADOR <operador_asignacion> <valor> ";"

<impresion> ::= "console" "." "log" "(" <lista_valores> ")" ";"

<condicional> ::= "if" "(" <condicion> ")" "{" <programa> "}" <opcional_else>

<opcional_else> ::= "else" "{" <programa> "}" | ε

<bucle_for> ::= "for" "(" <contenido_for> ")" "{" <programa> "}"

<bucle_while> ::= "while" "(" <condicion> ")" "{" <programa> "}"

<valor> ::= IDENTIFICADOR
           | LITERAL_NUMERICO
           | LITERAL_DECIMAL
           | LITERAL_CADENA
           | LITERAL_BOOLEANO

Conjuntos PRIMERO

No Terminal PRIMERO
<programa> { let, var, const, IDENTIFICADOR, console, if, for, while, ε }
<sentencia> { let, var, const, IDENTIFICADOR, console, if, for, while }
<declaracion> { let, var, const }
<tipo_variable> { let, var, const }
<asignacion> { IDENTIFICADOR }
<impresion> { console }
<condicional> { if }
<opcional_else> { else, ε }
<bucle_for> { for }
<bucle_while> { while }
<valor> { IDENTIFICADOR, LITERAL_NUMERICO, LITERAL_DECIMAL, LITERAL_CADENA, LITERAL_BOOLEANO }

Conjuntos SIGUIENTE

No Terminal SIGUIENTE
<programa> { $, } }
<sentencia> { let, var, const, IDENTIFICADOR, console, if, for, while, }, $ }
<declaracion> { let, var, const, IDENTIFICADOR, console, if, for, while, }, $ }
<asignacion> { let, var, const, IDENTIFICADOR, console, if, for, while, }, $ }
<impresion> { let, var, const, IDENTIFICADOR, console, if, for, while, }, $ }
<condicional> { let, var, const, IDENTIFICADOR, console, if, for, while, }, $ }
<opcional_else> { let, var, const, IDENTIFICADOR, console, if, for, while, }, $ }
<valor> { ;, ), , }

Según la tabla LL(1), el analizador puede seleccionar la producción correcta observando únicamente el primer token de entrada. Por ejemplo, si el token actual es if, se aplica la producción de condicional; si es console, se aplica la producción de impresión.
