3. Expresiones Regulares por Categoría Léxica

Las expresiones regulares permiten definir los patrones que utiliza el analizador léxico para reconocer cada tipo de token dentro del código fuente escrito en JavaScript.

Categoría Léxica Expresión Regular Ejemplo
Palabra Reservada \b(let|var|const|if|else|for|while|function|return|true|false)\b let, if, while
Identificador [a-zA-Z_áéíóúÁÉÍÓÚñÑ][a-zA-Z0-9_áéíóúÁÉÍÓÚñÑ]* edad, temperatura, contador1
Literal Numérico Entero \d+ 10, 25, 100
Literal Numérico Decimal \d+\.\d+ 36.5, 10.75
Literal de Cadena ".*?"|'.*?' "Hola", 'JavaScript'
Literal Booleano true|false true, false
Operador Aritmético \+|\-|\*|\/|\+\+|-- +, -, ++
Operador Relacional ==|!=|<=|>=|<|> ==, >=, <
Operador de Asignación =|\+=|-=|\*=|\/= =, +=, -=
Delimitadores ;|,|\. ;, ,, .
Agrupadores \(|\)|\{|\}|\[|\] ( ), { }, [ ]
Comentario de Línea //.* // Esto es un comentario

Expresión Regular Principal del Analizador Léxico

En la implementación del programa se utiliza una expresión regular principal para separar el código fuente en diferentes lexemas antes de clasificarlos como tokens.
